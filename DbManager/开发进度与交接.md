## 一、项目现状（一句话）
.NET 9 WPF 多数据库管理工具，**可正常启动运行**，全部代码 **0 错 0 警**。支持 6 种关系库（MySQL/MariaDB/SqlServer/PostgreSQL/Oracle/SQLite）完整功能；MongoDB/Redis/DB2 未实现（UI 已拦截）。

## 三、架构关键点
- **三层**：`DbManager.Common`（加密/日志/常量/错误翻译）、`DbManager.Core`（模型/接口/服务/**抽象层**/仓储）、`DbManager.Wpf`（MVVM 界面）。
- **跨库抽象层（Core/Abstractions + Core/Dialects）——所有跨库差异的唯一收敛点，禁止在 VM 里散写 `switch(dbType)`：**
  - `IIdentifierQuoter`：标识符引用（反引号/方括号/双引号）
  - `IDialect`：分页、限定名、时间/自增/拼接、**DDL（AddColumn/DropColumn/AlterColumn）**
  - `IDbTypeMapper` + `LogicalTypeEnum`：原生↔逻辑类型
  - `DialectProvider`：按 `DbTypeEnum` 取上述抽象
- **元数据**：`IDbMetadataService`（含 schema 维度）+ 5 库实现；`DbTreeNavigateService` 带 `MetadataCache`（懒加载+单节点刷新失效）。
- **本地存储**：SQLite（Dapper），`ConnRepository` 建表 + 字段迁移（迁移失败已容错，不阻断启动）。

## 八、已知限制（非 bug，已注释）
- SqlServer 删除列默认值未生成（需约束名）
- SQLite 不支持改列（注释提示，需重建表）
- 导出/复制仅当前页数据
- 连接串 SqlServer 未强制 Encrypt 设置（保持原语义，避免连接回归）
- 导出连接 JSON 含明文密码（跨机导入需要，属设计取舍）
- 新建表 SQLite 自增列内联为 `INTEGER PRIMARY KEY AUTOINCREMENT`（SQLite 限制，其余主键列不再单独约束）
- 类型化编辑：CLOB 因映射器归为二进制而只读（映射器既有行为，非本次改动）
- 事务：手动模式仅覆盖 SQL 编辑器（数据浏览编辑仍各自开关连接）；提交/回滚后自动开新事务
- SSH 隧道：按连接签名缓存复用，编辑连接参数后旧隧道需重启应用或删连接才释放（已提供 CloseForConnection 未在编辑流程接入）
