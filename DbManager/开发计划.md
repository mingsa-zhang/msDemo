# DbManager 重构开发计划

## Context
旧项目因 MaterialDesignThemes 5.x 样式兼容问题及累积修补导致启动失败，决定删除旧项目重新开发。基于执行计划.txt，去掉繁琐的单元测试，保留核心开发流程。

## 项目基础信息
- 项目名称：DbManager
- 框架：.NET 9 WPF + MVVM (CommunityToolkit.Mvvm)
- UI：MaterialDesignThemes 5.3.2 + AvalonEdit + EPPlus + CsvHelper
- 数据存储：本地 SQLite (Dapper)
- 支持数据库：MySQL、MariaDB、SQLServer、PostgreSQL、Oracle、SQLite、MongoDB、Redis、DB2

## 目录结构
```
DbManager/
├─ DbManager.Core/       # 核心库：模型、接口、服务、枚举、适配器
├─ DbManager.Common/     # 公共帮助类：加密、日志、常量、扩展
├─ DbManager.Wpf/        # WPF 主界面项目
```

> 去掉 DbManager.UnitTest 项目，核心逻辑通过启动运行验证

## MaterialDesignThemes 5.x 关键注意事项

1. **App.xaml** 只用 `BundledTheme`，不要手动添加 ResourceDictionary Source
2. **按钮样式**：不要使用 `MaterialDesignToolButton`、`MaterialDesignIconButton`、`MaterialDesignRaisedButton`、`MaterialDesignOutlinedButton` 等 4.x 样式名，5.x 中已移除
3. **安全做法**：按钮不指定 Style，使用默认隐式样式；或使用 `{DynamicResource}` 代替 `{StaticResource}`
4. **ColorZone、PackIcon、HintAssist** 等控件和附加属性在 5.x 中正常可用

## 编码规范
- 类/方法/属性：Pascal 大驼峰
- 私有字段：_开头 + 小驼峰
- 常量：全大写下划线
- View 零业务逻辑，全部下沉 ViewModel/Service
- 数据库操作统一参数化，禁止拼接 SQL
- 异步操作统一 async/await，禁止阻塞 UI
