# DbManager 重构开发计划（仿 Navicat 增强版）

## Context
旧项目因 MaterialDesignThemes 5.x 样式兼容问题及累积修补导致启动失败，决定删除旧项目重新开发。基于执行计划.txt，去掉繁琐的单元测试，保留核心开发流程。

> **本次修订说明（对标 Navicat）**：原计划把"链接 / 导航 / 呈现"当作并列功能阶段，缺少贯穿三段的跨库抽象层，且导航缺 Schema 层、呈现深度不足。本版新增 **第 0 章·跨库抽象层**、补齐 **Schema 层级**、深化 **数据呈现**、新增 **横切需求章**，并对连接会话做增强。
>
> 标记约定：🆕 本次新增需求　✳️ 本次修订/加强　✅ 当前代码已实现　🔲 待实现　⚠️ 已实现但不达标
>
> **可执行任务拆解见 [任务拆解.md](任务拆解.md)**（含文件映射、验收标准、依赖关系与迭代顺序）。

## 项目基础信息
- 项目名称：DbManager
- 框架：.NET 9 WPF + MVVM (CommunityToolkit.Mvvm)
- UI：MaterialDesignThemes 5.3.2 + AvalonEdit + EPPlus + CsvHelper
- 数据存储：本地 SQLite (Dapper)
- 支持数据库（关系型）：MySQL、MariaDB、SQLServer、PostgreSQL、Oracle、SQLite、DB2
- 支持数据库（非关系型，独立呈现范式）：MongoDB、Redis

## 目录结构
```
DbManager/
├─ DbManager.Core/       # 核心库：模型、接口、服务、枚举、适配器、抽象层
├─ DbManager.Common/     # 公共帮助类：加密、日志、常量、扩展
├─ DbManager.Wpf/        # WPF 主界面项目
```

> 去掉 DbManager.UnitTest 项目，核心逻辑通过启动运行验证

## 🆕 范围决策（MongoDB / Redis / DB2）
关系型 7 库（MySQL/MariaDB/SqlServer/PostgreSQL/Oracle/SQLite/DB2）共用 DataGrid + SQL 呈现范式。
- **MongoDB / Redis 不能套用 DataGrid**，必须单独设计呈现界面（见第六阶段·步骤 4/5）。
- 若资源有限，MongoDB/Redis/DB2 三者可作为 **P2 延后项**；但**枚举里已声明的类型，未实现前必须在"新增连接"界面禁用该类型**，禁止出现"能建连接却一展开就抛异常"的情况。

## 🆕 第 0 章：跨库抽象层设计（贯穿链接/导航/呈现的骨架）

> 目的：用抽象屏蔽 9 种库的差异，避免每个功能都写 9 路 `switch`。所有上层（导航、呈现、表设计）只依赖抽象，不依赖具体库。

### 0.1 方言层 `IDialect` / `DbSqlSyntaxAdapter`（✅ 已有雏形，✳️ 需统一为唯一入口）
- 分页（Limit/Offset、Top、ROWNUM、Fetch）、时间函数、自增语法、字符串拼接
- **要求**：数据浏览、SQL 执行、表设计的分页/语法生成**必须全部走此层**，禁止各 ViewModel 再自行拼接（当前 `DataBrowserViewModel` 重复实现了一套，需收敛）

### 0.2 🆕 标识符引用层 `IIdentifierQuoter`
- 统一封装 `` `名称` ``(MySQL) / `[名称]`(SqlServer) / `"名称"`(PG/Oracle/SQLite/DB2) 差异
- 全项目**唯一**加引号入口，杜绝硬编码反引号散落各处
- 提供 `Quote(name)`、`QuoteQualified(schema, table)` 等方法

### 0.3 🆕 类型映射层 `IDbTypeMapper`
- 数据库原生类型 ↔ 统一逻辑类型（文本 / 数值 / 日期时间 / 布尔 / 二进制 / JSON / 枚举）
- **驱动呈现层单元格编辑器选型**（日期选 DatePicker、JSON 选查看器、BLOB 选预览…）
- 表设计时提供各库可选类型下拉列表

### 0.4 🆕 对象元模型 `DbObjectModel`
- 统一实体：Schema、Table、View、Column、Index、ForeignKey、Trigger、Sequence、Procedure、Function
- 供导航树、表设计、DDL 生成共用（现仅有 `TableColumnModel`，需扩展成完整对象模型）

### 0.5 交付物
- `Abstractions/IDialect.cs`、`IIdentifierQuoter.cs`、`IDbTypeMapper.cs`
- `Models/DbObjectModel.cs`（含上述对象实体）
- 每种关系库提供各自实现，由工厂按 `DbTypeEnum` 分发

## MaterialDesignThemes 5.x 关键注意事项

1. **App.xaml** 只用 `BundledTheme`，不要手动添加 ResourceDictionary Source
2. **按钮样式**：不要使用 `MaterialDesignToolButton`、`MaterialDesignIconButton`、`MaterialDesignRaisedButton`、`MaterialDesignOutlinedButton` 等 4.x 样式名，5.x 中已移除
3. **安全做法**：按钮不指定 Style，使用默认隐式样式；或使用 `{DynamicResource}` 代替 `{StaticResource}`
4. **ColorZone、PackIcon、HintAssist** 等控件和附加属性在 5.x 中正常可用

## 编码规范
- 类/方法/属性：Pascal 大驼峰
- 私有字段：_开头 + 小驼峰
- 常量：全大写下划线
- View 零业务逻辑，全部下沉 ViewModel/Service
- 数据库操作统一参数化，禁止拼接 SQL（✳️ 表名/库名走引用层，条件走参数化）
- 异步操作统一 async/await，禁止阻塞 UI
- ✳️ 跨库差异一律走第 0 章抽象层，禁止在 ViewModel 内散写 `switch(dbType)`
