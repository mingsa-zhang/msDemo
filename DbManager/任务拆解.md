# DbManager 任务拆解（Backlog）

> 配套 [开发计划.md](开发计划.md)，把"仿 Navicat 增强版"的需求拆成可执行任务。
> 每个任务含：目标 / 涉及文件（🆕新建 ✏️修改）/ 验收标准 / 依赖。
> 状态：🔲待办　🚧进行中　✅完成

## 任务编号规则
`T-<Epic>.<序号>`，Epic 按依赖排序：A 地基 → B 导航 → C 呈现 → D 链接 → E 编辑器 → F 横切 → G 收尾。

## 依赖关系总览
```
A(抽象层地基)
 ├─> B(导航 Schema/对象)
 ├─> C(呈现深化)         ← 其中 C.2 多结果集 依赖 F.2 批量执行
 ├─> E(编辑器)           ← 其中 E.3 代码补全 依赖 B.4 缓存
 └─> G(表设计/收尾)
D(链接会话) ── 独立，可并行
F(横切) ── F.2 批量执行需先于 C.2；其余(事务/错误/性能)独立、贯穿全局
```

> ⚠️ 排期注意：**T-F.2 必须先于 T-C.2**，故 F.2 提前到 S3（不随 Epic F 整体压到 S5）。

## 建议迭代顺序（Sprint）
| Sprint | 内容 | 交付价值 |
|---|---|---|
| S1 | Epic A 全部 + T-C.1(参数化改造) | 地基就位，消除注入风险 |
| S2 | Epic B（Schema 层 + 对象补全 + 缓存） | 导航对标 Navicat |
| S3 | **T-F.2(批量执行，先行)** + Epic C（多结果集/类型化编辑器/筛选/表单/导出） | 呈现对标 Navicat |
| S4 | Epic D（SSH/会话）+ Epic E（编辑器补齐） | 连接与编辑体验 |
| S5 | Epic F 其余（事务/性能/错误）+ Epic G（收尾/禁用/警告） | 稳定与打磨 |

## Epic A：跨库抽象层地基 🔴

> 目标：所有跨库差异收敛到 Core/Abstractions，上层禁止散写 `switch(dbType)`。

### T-A.1 建立方言层 `IDialect` 🔲
- **目标**：把分页/Top/时间/自增/拼接语法从静态类升级为按库注入的方言接口
- **文件**：🆕 `DbManager.Core/Abstractions/IDialect.cs`；🆕 各库 `Dialects/MySqlDialect.cs` 等；✏️ `Adapters/DbSqlSyntaxAdapter.cs`（改为委托到 IDialect 或标记过时）
- **验收**：`IDialect.GetPagedSql/GetTopSql/CurrentTime/AutoIncrement/Concat` 可用；工厂按 `DbTypeEnum` 返回对应方言
- **依赖**：无

### T-A.2 标识符引用层 `IIdentifierQuoter` 🔲
- **目标**：统一 `` `` `` / `[]` / `""` 引用，全项目唯一入口
- **文件**：🆕 `DbManager.Core/Abstractions/IIdentifierQuoter.cs` + 各库实现
- **验收**：`Quote(name)`、`QuoteQualified(schema, table)`；grep 全项目无硬编码反引号/方括号拼接
- **依赖**：无

### T-A.3 类型映射层 `IDbTypeMapper` 🔲
- **目标**：原生类型 ↔ 逻辑类型（Text/Number/DateTime/Bool/Binary/Json/Enum），驱动单元格编辑器与字段下拉
- **文件**：🆕 `DbManager.Core/Abstractions/IDbTypeMapper.cs` + 各库实现；🆕 `Enums/LogicalTypeEnum.cs`
- **验收**：给定库+原生类型能返回逻辑类型；给定库能列出可选类型清单（供表设计）
- **依赖**：无

### T-A.4 对象元模型 `DbObjectModel` 🔲
- **目标**：统一对象实体，替代零散的 `TableColumnModel`
- **文件**：🆕 `DbManager.Core/Models/DbObjectModel.cs`（Schema/Table/View/Column/Index/ForeignKey/Trigger/Sequence/Procedure/Function）；✏️ `Models/TableColumnModel.cs`（并入或引用）
- **验收**：导航/表设计/DDL 生成共用同一套实体
- **依赖**：T-A.3（Column 引用逻辑类型）

### T-A.5 工厂接入抽象层 🔲
- **目标**：一个 `IDialectProvider` / 扩展现有工厂，统一按库拿 方言/引用/类型
- **文件**：✏️ `Services/DbMetadataServiceFactory.cs`、`DbExecuteServiceFactory.cs`；🆕 `Services/DialectProvider.cs`
- **验收**：上层通过工厂一次拿到某库的 dialect+quoter+typeMapper
- **依赖**：T-A.1/2/3
